Why this matters

OIDN's aux inputs are a contract, not a suggestion: albedo must stay linear because the network was trained on linear statistics, and normals must be exactly zero off-surface because that's the signal the network uses to recognize "nothing hit here — trust color alone." Violate either one and the network doesn't error — it just denoises worse, confidently, which is what makes these bugs easy to ship and hard to notice. See the generating aux inputs guide for the G-buffer recipes that avoid all three, and the gallery demo for aux inputs used correctly at scale.
