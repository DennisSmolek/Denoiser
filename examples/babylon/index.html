denoiser · babylon.js

Babylon renders a noisy PBR showcase; the denoiser cleans it — running on a different GPU device. Watch the two CPU copies that bridge them.

Two devices, one copy each way. Babylon's WebGPU engine can't adopt an external GPUDevice, and onnxruntime-web can't adopt Babylon's, so this is the honest CPU-copy integration: Babylon renders → RenderTargetTexture.readPixels() (GPU→CPU) → denoiser.denoise() (on the denoiser's own device) → RawTexture.update() (CPU→GPU) back into Babylon. The copies cost real time — which is why they're timed separately below — but the round-trip is still useful for stills and preview passes. See the custom-engines guide.
